# iOS Translation

Two-prompt strategy for translating the Flutter/Android camera library to native iOS/Swift.

## Target Stack

- iOS 26+, Swift 6, Metal 4
- SwiftUI + UIKit (UIViewRepresentable for MTKView)
- Swift-C++ direct interop (ObjC++ only if needed)
- OpenCV for CV processing (same as Android project)
- Swift 6 actor isolation (compile-time data race prevention)

## Workflow

```
1. Run setup.sh          → Packs codebase with repomix, copies reference docs
2. Add screenshots       → Place app screenshots in screenshots/
3. Run Prompt 1          → Cartographer audits the codebase → writes to output/
4. Review output/        → Verify completeness, correct gaps
5. Run Prompt 2          → Architect designs iOS app → writes to design/
6. Review design/        → Adjust phases, resolve open questions
7. Begin implementation  → Use Phase 1a from design/04-implementation-phases.md
```

## Documentation Format

All component documentation uses the L1-L5 layered format:

- **L1: Purpose** — Why it exists, architectural role
- **L2: Contracts** — Behavioral invariants (must be replicated on iOS)
- **L3: Decisions** — Design tradeoffs with rationale and source
- **L4: Edge Cases** — Guards, watchdogs, timing values with tuning history
- **L5: Android Reference** — Implementation mechanism (for context, not translation)

## Development Phases (iOS)

1a. Camera capture + state machine + lifecycle + permissions
1b. Camera controls (focus, AWB, AE, ISO, exposure, zoom)
2. Metal processing pipeline (replace raw preview with MTKView)
3. C++ / OpenCV sink integration + fan-out topology
4. Performance tuning + thermal/pressure resilience
5. Capture + recording (AVAssetWriter, EXIF)
6. Parity audit + polish

## Key Architecture Decisions

- **Sandwich pattern**: SwiftUI (top) → UIKit bridge (middle) → Engine (bottom)
- **Compile-time isolation**: @MainActor for UI, @globalActor for ML, serial queue for camera
- **Zero-copy pipeline**: CVPixelBuffer → MTLTexture (via cache), CVPixelBuffer → cv::Mat (via pointer)
- **Back-pressure**: AsyncStream with .bufferingNewest(1) — drop old frames automatically
- **Sendable**: Buffers stay on one queue, only result structs cross actor boundaries
- **VTFrameProcessor**: Evaluate for standard transforms before writing custom Metal shaders
